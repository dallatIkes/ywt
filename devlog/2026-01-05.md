# Devlog #2 - 2026-01-05

## What I did

- Refactored the **WHOLE** project (database, operations, helper functions, models) to use the username as unique identifier
- Implemented the authentication with jwt
- Added a `.env` file to store the secret key used to generate tokens
- Added the `/token` and `/users/me` endpoints
- Created a security folder for helpers
- The `/users/me` endpoint sends everything we need for the app (username, recommended_to_me, recommended_by_me)
- Added thee reload option to `run_server()` 
- Added a new helper to get a user with the ID (not the one used by the endpoint which use the username)

## What I learned

- It's better to use the username as much as possible rather than the ID
- The ID must still be used as foreign key (because if we change a username we break the relationship)
- For some reason, when we activate the reload option with uvicorn, we must change the `app` variable in the `run` method by the `"main:app"` string
- The whole token workflow breaks if we don't specifically use `datetime.utcnow()` (which is weird because the method is now deprecated...)
- If a python function raises an Exception, it's not considered as returning `None`, it just doesn't return anything
- `return` === `return None`
